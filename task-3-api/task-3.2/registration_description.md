# Алгоритм создания пользователя
## 1.Backend получает HTTP POST запрос /api/users/register

## 2.Сервис проверяет наличие обязательных полей:

  - firstName
  - lastName
  - Email
  - Password
  - confirmPassword

## 3. Сервис проверяет правильность написания Email

## 4. Если Email написан неверно:
  - сервис возвращает ошибку 400 Bad Request
## 5. Сервис проверяет что длина Password > 8

## 6. Если длинна Password <= 8:
  - сервис возвращает ошибку 400 Bad Request
    
## 7. Сервис проверяет Password = confirmPassword

## 8. Если Password != confirmPassword:
  - сервис возвращает ошибку 400 Bad Request

## 9. Сервис выполняет поиск пользователя в БД по email

## 10.Если пользователь с таким email уже существует:
  - сервис возвращает ошибку 409 Conflict

## 11. Если проверки пройдены успешно:
  - пароль хешируется
  - создается новый пользователь в БД

## 12.Сервис возвращает успешный ответ:
  - HTTP статус 201 Created
  - идентификатор пользователя
  - сообщение об успешной регистрации
